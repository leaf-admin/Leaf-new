## 📊 GATILHOS PARA LIBERAÇÃO DE CONVITES

### **Sinais que indicam necessidade de mais motoristas:**

#### **1. ETA Médio > 8 minutos**
```
Se o tempo médio de espera ultrapassar 8 minutos, 
é sinal de que há mais demanda que oferta.
Ação: Liberar +50 motoristas
```

#### **2. Taxa de Aceite < 70%**
```
Se menos de 70% das corridas são aceitas,
motoristas estão sobrecarregados ou muito distantes.
Ação: Liberar +50 motoristas
```

#### **3. Taxa de Cancelamento > 15%**
```
Se passageiros cancelam muito (> 15%),
provavelmente estão esperando demais.
Ação: Liberar +50 motoristas
```

#### **4. Tempo Médio de Espera por Aceite > 3 minutos**
```
Se demora mais de 3 minutos para uma corrida ser aceita,
há escassez de motoristas disponíveis.
Ação: Liberar +50 motoristas
```

#### **5. Pico de Demanda Consistente (3 dias seguidos)**
```
Se há pico de demanda por 3 dias consecutivos,
não é pontual, é tendência.
Ação: Liberar +100 motoristas
```

## 📈 COMPARAÇÃO: ESTRATÉGIA CONSERVADORA vs AGRESSIVA

### **Estratégia Conservadora (675 motoristas mês 1):**
```
✅ Cobertura completa desde o início
✅ ETA bom desde o início (5-7 min)
❌ Investimento inicial alto
❌ Risco de subutilização em regiões de baixa demanda
```

### **Estratégia Agressiva (200 motoristas mês 1):**
```
✅ Investimento inicial baixo
✅ Escala baseada em demanda real
✅ Menor risco de subutilização
❌ ETA inicial pode ser alto (6-8 min)
❌ Cobertura gradual (não 100% desde início)
```

## ⚠️ RISCOS E MITIGAÇÕES

### **Risco 1: ETA Alto no Início (6-8 min)**

**Mitigação:**
- Começar em 1 região estratégica (centro, maior demanda)
- Comunicar claramente que é fase inicial
- Bonificação para primeiros motoristas (garantir mais online)
- Monitorar ETA em tempo real

### **Risco 2: Demanda Cresce Mais Rápido que Oferta**

**Mitigação:**
- Ter lista de espera de motoristas pré-cadastrados
- Processo de onboarding rápido (7 dias)
- Liberar convites em lotes (50-100 por vez)
- Alertar automaticamente quando gatilhos são ativados

### **Risco 3: Passageiros de Regiões Não Cobertas Cancelam**

**Mitigação:**
- Expansão rápida quando houver demanda comprovada
- Sistema de waitlist por região
- Notificar quando região for coberta
- Priorizar expansão para áreas com maior demanda
